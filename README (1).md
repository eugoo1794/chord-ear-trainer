# Chord Ear Trainer

和音を聞いてコードの種類を当てる、耳トレ用のWebアプリ。
間違えたときに、その場で正解と自分の誤答を **鳴らし比べ** できるのが特徴。

👉 **[アプリを開く](https://eugoo1794.github.io/chord-ear-trainer/)**

## 使い方

1. 「▶ 再生」で和音が鳴る（Space キーでも再生・リプレイ）
2. 聞こえたコードの種類（quality）を選ぶ
3. 答え合わせ後、選択肢が **試聴ボタン** に変わる → タップで聴き比べ
4. 「次の問題」または Enter キーで次へ

## できること

- 出題範囲を自分で選択（トライアド / 7th / テンション）
- ルートは「ランダム / C固定」をワンタップ切り替え
- 「クオリティだけ当てる」／「ルートも含めたフルネームを当てる」の2モード
- 同時（ブロック）・分散（アルペジオ）の鳴らし分け
- 正答率・連続正解・苦手コードの集計
- オフライン対応（Service Worker）／スマホはホーム画面に追加でアプリ化

## 技術メモ

- 単一の `index.html`（依存ライブラリなし）
- 音は Web Audio API で生成（音源ファイル不要）
- PWA構成：`manifest.json` + `sw.js` + アイコン

## ファイル構成

```
index.html      本体
manifest.json   PWA設定
sw.js           オフライン用 Service Worker
icon-192.png    アイコン
icon-512.png    アイコン
```

## ローカルで動かす

`index.html` をブラウザで開くだけでも動く。
Service Worker を含めて確認したい場合は、簡易サーバー経由で：

```powershell
# このフォルダで
python -m http.server 8000
# → http://localhost:8000 を開く
```

## 更新するとき

コードを直しても反映されないときは、`sw.js` のキャッシュ名
（`chord-ear-v1`）の数字を上げてから再デプロイすると確実に更新される。
